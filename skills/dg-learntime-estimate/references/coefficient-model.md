# 估算系数模型

dg-learntime-estimate 的完整系数表 + 依据 + 适用场景。SKILL.md 引用此文件，Claude 按此表算基础值。

## 基础系数（按输入类型）

### 文本类（Markdown / 纯文本）

| 内容语言 | 系数 | 依据 |
|---------|------|------|
| 中文技术文 | **350 字/分** | 教育部《义务教育语文课程标准》成人默读标准 300-500 字/分，技术文含术语、概念密度高，取中位偏下 |
| 英文技术文 | **200 词/分** | Brysbaert 2019（University of Ghent）成人平均阅读 238 词/分；技术文降速至 200 |

**计算**：`基础值（分） = 字数 ÷ 系数`

字数判定规则：
- 中文字符：每个汉字 + 标点计 1
- 英文词：按空格切分的 token 数
- 中英混排：按主要语言归类（占比 > 60% 决定）

### 代码

| 模式 | 系数 | 依据 |
|------|------|------|
| 精读（含跑 demo / 调试 / 对照文档） | **50 行/时** | 经验值；精读含调试、对照文档、回查 |
| 通读（理解结构） | **150 行/时** | 经验值，约精读的 3 倍 |

**计算**：`基础值（分） = 行数 ÷ 系数 × 60`

精读 vs 通读判定启发式（由 Claude 判断）：
- **精读**：核心模块 / 算法 / 安全相关 / 入口文件（main / index）
- **通读**：配置 / 胶水代码 / 样式 / 测试 fixture / 工具函数

### PDF

**优先按字数算**（页数浮动太大）：

| PDF 类型 | 字数系数 | 依据 |
|---------|---------|------|
| 技术 PDF（论文 / 规范 / 白皮书） | 250 中文字/分 或 150 英文词/分 | 比 Markdown 慢 30%（公式、图表、来回翻阅） |
| 普通文档 | 同文本类基础系数 | 经验值 |

**字数提取失败时**用页数 fallback：

| PDF 类型 | 页数系数 | 依据 |
|---------|---------|------|
| 技术 PDF | **15 分/页** | 假设每页 500 词 × 阅读速度，含公式回看 |
| 普通文档 | **5 分/页** | 经验值 |

### 书籍

**输入**：用户给 `书名 + 页数 + 类型`。

| 类型 | 系数 | 依据 |
|------|------|------|
| 技术书 | **3 分/页** | 行业经验值（含代码、回查、跑示例） |
| 教材 | **5 分/页** | 含习题、推导、概念深挖 |
| 小说 / 通识 | **1 分/页** | 速读经验值 |

**计算**：`基础值（分） = 页数 × 系数`

**类型缺省时由 Claude 判断**（基于书名或上下文）：
- 含「算法」「编程」「架构」「设计」等 → 技术书
- 含「教程」「导论」「原理」等 → 教材
- 含「小说」「史」「传记」等 → 通识

## 视频时长（合集 + 单视频统一）

视频输入（手报 / yt-dlp 抓取）**直接按时长作为基础值**——不区分纯看 / 含练习 / 高难度，不应用内在难度调整。

**计算**：
- 单视频：`基础值（分） = 视频时长（分）`
- 合集（多 P）：`base_total = Σ 每 P 时长（分）`（简单累加，跟文件夹聚合一致）

**依据**：原 v2.8.0 模型的视频倍率（1.0/1.5/2.0）+ 内在难度（×0.9/1.0/1.3）对合集场景判断成本高（每个 P 都要判），且三水平倍率（1.8/1.0/0.6）已能覆盖大部分差异。v2.8.1 起视频模式简化为纯时长，模型更透明，用户能心算验证。

**合集探测**：用 `yt-dlp --print "%(title)s|%(duration)s" <URL>`（不带 `--no-playlist`）拿所有 P 的 title + duration（秒）。
- 输出 1 行 → 单视频
- 输出 ≥ 2 行 → 合集
- 输出 0 行（空） → 完全失败

## 水平倍率（输出时同时列三档，不让用户选）

每份资料的合计同时输出 3 个水平的估时（基础合计 × 对应倍率），用户自己认领：

| 水平 | 倍率 | 适用人群 |
|------|------|---------|
| 零基础 | **1.8×** | 首次接触该领域，需来回查资料、概念陌生、跑不通代码 |
| 有基础 | **1.0×** | 同领域有相关经验（基础值即此水平） |
| 熟练 | **0.6×** | 同领域核心熟练，扫描重点、跳过基础章节 |

**依据**：3 倍量级差异反映真实学习场景。同一份《Designing Data-Intensive Applications》对零基础读者可能需要 60 小时，对熟练架构师 20 小时——这种差异远大于"难度倍率"（0.8~1.4）能表达的。

**计算**：`最终估时 = base_total × 水平倍率`

水平倍率**不让用户指定**——用户对自己不熟悉的资料容易误判水平（"我觉得我懂"和实际熟练度差距大）。同时输出 3 个数字让用户事后回头看，能感受到自己水平的真实位置。

## 文件夹聚合

**简单累加**——所有文件基础估时相加得到 `base_total`：

```
base_total = Σ (每个文件的基础值)
```

**不引入疲劳衰减**（连续学习的效率下降是真实存在的现象，但量化系数是经验值，且短估时场景下用不上，引入会让输出多一行不透明的修正项）。

如果用户反馈"连续学 N 小时实际更慢"，未来可通过 `--fatigue` flag 加回累进倍率（见 SKILL.md Extension Support）。

## 资料内在难度（Claude 在算基础值时调整）

**由 Claude 基于内容判断**，**不让用户指定**（用户对自己不熟悉的资料容易低估难度）：

| 内在难度 | 倍率 | 适用场景 |
|---------|------|---------|
| 难 | **×1.3** | 数学密集 / 跨领域 / 前沿论文 / 含大量证明推导 |
| 中 | **×1.0** | 普通（默认） |
| 易 | **×0.9** | 已熟悉主题 / 入门资料 / 流行科普 |

**应用顺序**：
1. 先算基础值（量化信号 ÷ 系数）
2. **视频模式跳过此步**（不应用内在难度调整，见上方「视频时长」段）
3. 其他模式（文本 / PDF / 书）：再乘内在难度倍率（×1.3 或 ×0.9）
4. 得到调整后的基础值
5. 最后乘水平倍率（×1.8 / ×1.0 / ×0.6）

**判断启发式**：
- 数学密集：含大量公式、定理、证明（数学/统计/物理/ML 理论）
- 跨领域：读者 assumed knowledge 跨多个领域（如生物信息学、量子计算入门）
- 前沿论文：含未普及的术语、依赖最新研究背景
- 已熟悉：用户领域内的标准教材 / review 文章
- 入门资料：写给完全新手的教程 / "X for Dummies" 风格

## 完整计算示例

### 示例 1：单文件估时

输入：`README.md`（4,200 中文字，普通技术文档）

1. 基础值 = 4200 ÷ 350 = 12 分
2. 内在难度：普通 → ×1.0 → 12 分
3. base_total = 12 分
4. 三水平：
   - 零基础 = 12 × 1.8 = 22 分
   - 有基础 = 12 × 1.0 = 12 分
   - 熟练 = 12 × 0.6 = 7 分

输出时间格式：3 个都 < 60 分，只显示分。

### 示例 2：文件夹估时

输入：`tutorials/` 含 5 个 md（共 21,000 中文字）+ 1 个 py（300 行精读，数学密集算法）

1. md 基础值 = 21000 ÷ 350 = 60 分
2. py 基础值 = 300 ÷ 50 × 60 = 360 分
3. 内在难度：
   - md 普通 → 60 × 1.0 = 60 分
   - py 数学密集 → 360 × 1.3 = 468 分
4. base_total = 60 + 468 = 528 分 = 8 时 48 分
5. 三水平：
   - 零基础 = 528 × 1.8 = 950 分 = 15 时 50 分
   - 有基础 = 528 × 1.0 = 528 分 = 8 时 48 分
   - 熟练 = 528 × 0.6 = 317 分 = 5 时 17 分

### 示例 3：视频估时（单视频，手报）

输入：`这个视频 45 分钟`

1. 基础值 = 45 分（视频时长直接作为基础值）
2. 视频模式跳过内在难度调整
3. base_total = 45 分
4. 三水平：
   - 零基础 = 45 × 1.8 = 81 分 = 1 时 21 分
   - 有基础 = 45 × 1.0 = 45 分
   - 熟练 = 45 × 0.6 = 27 分

### 示例 4：书籍估时

输入：`DDIA 400 页 技术书`

1. 基础值 = 400 × 3 = 1200 分
2. 内在难度：跨领域（分布式系统 + 数据库 + 系统设计）→ ×1.3 → 1560 分
3. base_total = 1560 分 = 26 时
4. 三水平：
   - 零基础 = 1560 × 1.8 = 2808 分 = 46 时 48 分
   - 有基础 = 1560 × 1.0 = 1560 分 = 26 时
   - 熟练 = 1560 × 0.6 = 936 分 = 15 时 36 分

### 示例 5：合集视频估时（B 站多 P 课程）

输入：`https://www.bilibili.com/video/BV1zv5561EtZ`（吴恩达 AI Prompting，18 P，总时长约 130 分）

1. `yt-dlp --print "%(title)s|%(duration)s" <URL>` 拿到 18 行 title|duration（秒）
2. 每个 P 时长直接作为基础值（不乘倍率、不调难度）
3. base_total = 18 个 P 时长累加 = 约 130 分 = 2 时 10 分
4. 三水平：
   - 零基础 = 130 × 1.8 = 234 分 = 3 时 54 分
   - 有基础 = 130 × 1.0 = 130 分 = 2 时 10 分
   - 熟练 = 130 × 0.6 = 78 分 = 1 时 18 分
