# Subagent 任务模板

dg-learn 处理多资料时，每个资料派一个 general-purpose subagent。本文是 subagent 的任务说明 + 输出模板，主 skill 用本文格式 dispatch subagent，subagent 必须严格按模板返回。

## 为什么用 subagent

- **保护主 skill context**：原资料全文不进主 skill context，避免撑爆
- **并行处理**：多份资料同时处理，速度快
- **隔离失败**：某份资料处理失败不影响其他资料

## subagent 类型选择

**用 general-purpose subagent，不用 Explore agent**。

| 类型 | 是否适合 dg-learn | 理由 |
|------|------------------|------|
| `general-purpose` | ✅ 用这个 | 能完整读取文件、深度分析、按模板输出 |
| `Explore` | ❌ 不用 | 官方描述：「reads excerpts rather than whole files and will miss content past its read window」——会丢内容，不适合处理需要完整理解的资料 |

## 并行上限

**最多 5 个 subagent 同时跑**。资料数 > 5 时分批：
- 第 1 批：资料 1-5 派 5 个 subagent
- 第 2 批：资料 6-10 派 5 个 subagent
- 以此类推

## 主 skill dispatch subagent 的方式

主 skill（dg-learn）调用 `Agent` 工具，参数：
- `subagent_type: "general-purpose"`
- `description: "提取资料摘要：{资料名}"`
- `prompt`: 用下面的模板拼接

## Subagent Prompt 模板

主 skill 给每个 subagent 派任务时，用以下 prompt（替换 `{占位符}`）：

```
你是 dg-learn skill 的资料处理 subagent。任务是读取一份学习资料，提取结构化摘要，按指定模板返回。

## 资料信息

- 资料标识：{资料序号，如 #1}
- 资料类型：{md | pdf | url}
- 资料来源：{文件相对路径 或 URL}

## Step 1：字数检查

读取前先用以下命令估算字数（PDF 用页数估算）：

- md/markdown 文件：`wc -m {文件路径}`
- PDF 文件：用 Read 工具读取后估算（一般按每页 500-800 字）
- URL：用 WebFetch / mcp__web_reader__webReader 抓取后估算

**单文件字数阈值：15 万字**。如果资料超过此阈值，**不要继续处理**，返回如下错误：
```
ERROR: 资料超过字数阈值
资料：{来源}
字数：{N} 万字
阈值：15 万字
建议：请用户拆分资料后再用
```

主 skill 看到这个错误后会在最终报告里提示用户拆分。

## Step 2：读取资料

字数检查通过后，用合适的方式读取：
- md/markdown：Read 工具
- PDF：Read 工具（支持 PDF）
- URL：WebFetch 或 mcp__web_reader__webReader

读取失败时（网络问题、文件不存在、PDF 解析失败等）返回：
```
ERROR: 资料读取失败
资料：{来源}
错误：{失败原因}
```

主 skill 看到这个错误后会在最终报告里提示用户。

## Step 3：提取摘要

读完资料后，按以下模板提取结构化摘要。**严格按模板格式输出**，便于主 skill 合并。

```markdown
## 资料：{资料名}
- 类型：{md | pdf | url}
- 来源：{相对路径 或 URL}
- 字数：{N}
- 主题：{一句话概括资料讲什么}

### 核心知识点
（列出所有值得作为"考点"的概念，每个配 1-2 句解释）

1. **{知识点名}**：{1-2 句解释}
2. **{知识点名}**：{1-2 句解释}
...

### 重点
（资料里强调的、重要的内容，简要说明为什么重要）

- **{重点项目}**：{为什么重要}
- **{重点项目}**：{为什么重要}
...

### 难点
（资料里难懂、容易混淆、需要花时间理解的内容）

- **{难点项目}**：{难在哪里}
- **{难点项目}**：{难在哪里}
...

### 关键示例 / 代码片段
（资料里的重要示例、代码、图示说明，3-5 个最有代表性的）

- 示例 1：{简要描述 + 代码/示意}
- 示例 2：{简要描述 + 代码/示意}
...

### 建议出题点
（建议主 skill 哪些点适合出什么类型的题）

- 适合做选择题的：{哪些知识点适合出选择/判断}
- 适合做简答题的：{哪些知识点适合出简答}
- 适合做应用题的：{哪些知识点适合出应用/实战}
```

## 重要约束

1. **只读取这一份资料**——不要尝试读取其他资料（那是其他 subagent 的工作）
2. **按字面模板输出**——不要改变段落标题、不要增减段落、不要换个写法
3. **核心知识点用编号列表**——便于主 skill 合并去重
4. **不要生成题目**——题目由主 skill 综合 N 份摘要后统一生成（避免重复和遗漏）
5. **不输出与本任务无关的内容**（如"我已读完资料，开始处理..."这种过程性叙述）

## 输出长度建议

- 短资料（< 2000 字）：摘要 200-500 字
- 中等资料（2000-10000 字）：摘要 500-1500 字
- 长资料（10000-150000 字）：摘要 1500-3000 字

不要把整个资料原样塞进摘要——摘要是"浓缩的结构化提取"，不是"全文复制"。
